Smeetz Elements SDK Prototype

Flux de réservation inline immersif

Plus de lightbox — le tunnel de réservation s'intègre directement dans la page produit. 100 % brandé, 100 % à votre design system. Idéal pour la billetterie principale du parc.

🧩

Prototype 1 — Inline Booking Flow

Cet emplacement accueillera le composant Smeetz Elements généré via le skill smeetz-elements-booking-flow-creation. Le tunnel complet (sélection date · participants · paiement) s'affiche directement ici, sans modal.

<!-- Smeetz Elements SDK embed point -->
<div id="smeetz-booking-inline"></div>
<script src="https://elements.smeetz.com/sdk.js"></script>
<script>
  SmeetzElements.mount('#smeetz-booking-inline', {
    productId: '76037', // Entrée Parc
    mode: 'inline',
    theme: 'natae-light'
  });
</script>
🎯 Pourquoi montrer ce flux à Nataé

Démontre que Smeetz peut aller au-delà du widget standard : composition 100 % custom, intégration native dans le site existant, expérience utilisateur sans rupture. Différenciateur fort vs Qweekle (widget-only).

Cas d'usage

Pour quelle situation ce flux ?

⚡

Conversion maximale

Pas de clic supplémentaire pour ouvrir un modal — l'utilisateur voit le tunnel dès l'arrivée sur la page. Réduit le drop-off entre intention et achat.

🎨

Design 100 % cohérent

Hérite des fonts, couleurs et composants de votre design system. Le tunnel respire le même air que le reste du site.

📱

Mobile-first

Composants Smeetz Elements optimisés mobile (60 % du trafic Nataé). Pas de modal qui prend tout l'écran sur smartphone.

Widget vs SDK

Quand choisir l'un ou l'autre

Critère Widget SDK Elements
Temps de mise en place ⚡ Quelques minutes 🛠️ Quelques jours
Personnalisation visuelle Limitée aux couleurs ✨ Totale
Intégration site existant Modal/lightbox Inline natif
Mise à jour produits Auto (back-office) Auto (back-office)
Idéal pour Lancement rapide Site premium / haute conversion
